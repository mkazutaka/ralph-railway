# Workflow Editor

ワークフローを視覚的に組み立て、ノード単位で設定し、実行・テストする編集体験。

## キャンバスの基本操作

- ユーザとして、キャンバス上にワークフローのノードと接続が視覚的に配置されているのを見たい
- ユーザとして、キャンバスをドラッグして全体をパンしたい
- ユーザとして、ズームコントロールでキャンバスを拡大・縮小したい
- ユーザとして、ズーム率を100%（フィット）にワンクリックで戻したい
- ユーザとして、キャンバス全体の俯瞰をミニマップで把握し、現在の表示範囲を確認したい
- ユーザとして、ミニマップの該当箇所をクリックしてその位置にジャンプしたい
- ユーザとして、キャンバス上に背景のドットグリッドが表示されてノード位置の目安になってほしい
- 初めてのユーザとして、キャンバス下部のヒントから操作方法（ドラッグでパン、スクロールでズームなど）を確認したい

## ノードの追加

- ユーザとして、キャンバス右上のFAB（Add Nodeボタン）から新しいノードを追加したい
- ユーザとして、ノード追加ポップアップでノードの種別（Manual Run / Shell / For Loop / Claude Implement / Claude Review など）を選びたい
- ユーザとして、ノードの出力ハンドルから線をドラッグして次のノードを追加するショートカットを使いたい
- ユーザとして、対応していないノード種別を選んだときにエラーメッセージで知りたい

## ノードの操作

- ユーザとして、ノードをドラッグしてキャンバス上の位置を変えたい
- ユーザとして、ノードをクリックして選択し、右パネルでプロパティを編集したい
- ユーザとして、選択中のノードがハイライトされ、他のノードと区別できるようにしたい
- ユーザとして、ノードを削除するときは確認のうえ実行したい
- ユーザとして、複数ノードをまとめて選択・移動・削除したい
- ユーザとして、ノードをコピー＆ペーストして同じ設定を複製したい

## ノードの種別表示

- ユーザとして、ノードの種別ごとに色（trigger / action / logic / ai など）が分かれていて区別したい
- ユーザとして、各ノードのアイコンとタイトルでひと目で役割を理解したい
- ユーザとして、ループ等のコンテナノードはネストされた子タスクを内側に視覚的に内包して表示してほしい

## 接続（エッジ）

- ユーザとして、ノードの出力ハンドルから別ノードの入力ハンドルへドラッグして接続を作成したい
- ユーザとして、接続が成立したらハンドルとエッジが強調表示されて成功が分かるようにしたい
- ユーザとして、ループのdo[0]・do[1]などの分岐ラベルがエッジ上に表示されて分岐先が分かりたい
- ユーザとして、無効な接続（循環、型不一致、出力→出力など）はその場で拒否されて理由を伝えてほしい
- ユーザとして、既存のエッジをクリック・選択して削除したい

## キャンバスツールバー

- ユーザとして、キャンバス上部のツールバーから選択／パン／整列／グリッドスナップなどの操作モードを切り替えたい
- ユーザとして、Undo / Redoでキャンバス上の操作をやり直したい
- ユーザとして、Auto-layoutで現在のノード配置を整列させたい

## YAMLとの双方向同期

- ユーザとして、YAMLエディタの変更がキャンバスに即座に反映されるのを見たい
- ユーザとして、キャンバス上の操作（追加・移動・削除・接続）がYAMLに反映されるのを見たい
- ユーザとして、YAMLが構文エラーのときはキャンバスを最後の有効状態に保ち、エラー内容をエディタ近くに表示してほしい
- ユーザとして、YAMLでサポートされていない構造のときはキャンバス側で警告したい

## ノードインスペクタ（右パネル）

- ユーザとして、キャンバスでノードを選択すると右パネルにそのノードの設定が表示されてほしい
- ユーザとして、何も選択していないときは右パネルでワークフロー全体の設定（メタ情報、入力、グローバル変数）を編集したい
- ユーザとして、複数ノードを選択したときは共通項目だけを編集できるようにしたい
- ユーザとして、右パネルのヘッダーで現在編集中のノード名・種別・アイコンを確認したい
- ユーザとして、右パネル上部のタブで「Config / Inputs / Outputs / Logs」など編集対象を切り替えたい
- ユーザとして、未入力の必須項目がある場合はそのタブにバッジ（!）が付いて気付きたい
- ユーザとして、ノード固有のフィールド（command, prompt, condition など）をフォームで編集したい
- ユーザとして、変更内容は即座にYAMLとキャンバスに反映されてほしい
- ユーザとして、入力値のバリデーションエラー（必須未入力、型不一致、無効なjq式）をフィールド近くに表示してほしい
- ユーザとして、シェルコマンドや式の入力ではコードエディタ風（モノスペース・シンタックスハイライト）で編集したい
- ユーザとして、変更を破棄して直前の状態に戻すボタンが欲しい
- ユーザとして、上流ノードの出力を参照する変数（例: $tasks.previous.output.foo）をオートコンプリートで挿入したい
- ユーザとして、参照先のノードがいま存在するか・有効かをその場で検証してほしい
- ユーザとして、出力スキーマ（型・サンプル値）を確認したい

## 実行とテスト

- ユーザとして、トップバーのRunボタンから現在のワークフローを実行したい
- ユーザとして、実行中はRunボタンが「Running…」「Stop」など状態に応じた表示になってほしい
- ユーザとして、未保存の変更がある状態でRunを押したら、保存してから実行するか変更を破棄するか確認してほしい
- ユーザとして、構文エラーがあるYAMLでは実行を開始できず、原因を提示してほしい
- ユーザとして、実行中のノードがキャンバス上で進行中（パルスやアクセントカラー）として表示されるのを見たい
- ユーザとして、完了済み・失敗・スキップ・実行待ちの各ノード状態を色や記号で区別したい
- ユーザとして、トップバーのStatus Badgeで全体の実行状況（idle / running / success / failed）を確認したい
- ユーザとして、キャンバス下部のExecution Barで現在のステップ・経過時間・進捗を確認したい
- ユーザとして、失敗したノードをクリックしてエラーログとスタックトレースを参照したい
- ユーザとして、実行を途中で停止したい
- ユーザとして、停止後に実行が完全にキャンセルされたことを確認したい
- ユーザとして、特定のノードからのみ部分実行（再実行）したい
- 権限のないユーザとして、Runボタンを押したときに実行権限が必要であるとフィードバックしてほしい
- ユーザとして、左サイドバーのRecent Runsから直近の実行を一覧したい
- ユーザとして、各実行の開始時刻・所要時間・成否がリストでわかりたい
- ユーザとして、Recent Runsから過去の実行を選んで結果を再表示したい
- ユーザとして、各ノードの標準出力・エラー出力をリアルタイムで見たい
- ユーザとして、ノードの出力（.output.<name>）を右パネルで確認したい
- ユーザとして、ログをコピー・ダウンロードして外部に共有したい
- ユーザとして、ログ量が多い場合は折りたたみと検索ができるようにしてほしい
- ユーザとして、右パネル下部のTest Sectionから選択中ノードだけを単独でテスト実行したい
- ユーザとして、テスト用のダミー入力値を編集して保存したい
- ユーザとして、テスト結果（出力値・所要時間・ログ）を右パネル内に表示してほしい
- ユーザとして、テスト実行が失敗したときはエラー内容・原因の手がかりを表示してほしい
- 実行権限がないユーザとして、テストボタンを押した際に権限不足である旨を表示してほしい

## パターンの挿入

- ユーザとして、対応しているワークフローパターンの一覧（do / if / switch / fork / loop / try-catch / retry / set など）をカード形式で見たい
- ユーザとして、各パターンに簡潔な説明とYAMLサンプル・ミニ図解が添えられていてほしい
- ユーザとして、`do` ブロックが順次実行のシーケンスであることを図とコードで理解したい
- ユーザとして、`if` ガードで条件が偽のときにタスクがスキップされる挙動を理解したい
- ユーザとして、`switch` で条件に応じて指定タスクへルーティングされる挙動を理解したい
- ユーザとして、`fork` で複数ブランチが並列実行され出力がマージされる挙動を理解したい
- ユーザとして、`loop` (Dify-style) と `for.do` の違いを比較したい
- ユーザとして、`try / catch` で失敗時の代替経路を定義できることを理解したい
- ユーザとして、`retry` で指数バックオフ付きリトライを設定できることを理解したい
- ユーザとして、`set` で `.output.<name>` に値やjq式を割り当てられることを理解したい
- ユーザとして、パターンカードから「このパターンを挿入」ボタンで現在編集中のワークフローに即挿入したい
- ユーザとして、サンプルYAMLをワンクリックでクリップボードにコピーしたい
- ユーザとして、サンプルを挿入したときに既存ノードと衝突しないよう新規IDが割り当てられてほしい
- ユーザとして、サポートされていない構文をサンプルに含むパターンは「未サポート」として明示してほしい
- ユーザとして、検索・タグでパターンを絞り込みたい（制御フロー / 並列 / エラー処理 / 変数 など）
